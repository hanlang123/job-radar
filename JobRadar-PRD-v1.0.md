## 1. 文档说明

### 1.1 项目背景

当前前端求职市场竞争激烈，求职者面临以下痛点：岗位信息分散在各大平台，难以宏观把握市场趋势；JD 中的技术要求需要人工逐条解读，效率低下；自身技能与市场需求之间的差距缺乏量化手段；技术栈之间的关联关系（如"要求 Vue 的岗位有多少同时要求 TypeScript"）无法直观获取。

JobRadar 旨在通过数据化、可视化、AI 智能化三重手段，将爬取的 Boss 直聘岗位数据转化为可操作的求职洞察。

### 1.2 项目定位

面向前端开发者的职位情报分析平台。不是招聘网站，不提供投递功能，核心价值是"分析"而非"连接"。

### 1.3 目标用户

前端开发者（初中级为主），正在求职或准备求职，希望了解市场行情、明确技能差距、获取 AI 辅助的求职建议。

### 1.4 配套文档

本文档为产品需求文档，配套的技术实施文档包括：「JobRadar 技术方案 v1.0」（系统架构、模块设计、数据库 Schema、实施计划）和「CLAUDE-CODE-PROMPT.md」（开发执行指令）。

## 2. 产品目标

### 2.1 核心目标

为求职者提供一站式的岗位市场分析工具，涵盖数据总览、技术趋势洞察、AI 智能分析三大能力。

### 2.2 技术展示目标

本项目同时作为个人求职作品集项目，需充分展示以下技术能力：基于 Nuxt 3 的 SSR 混合渲染策略、自研 vue-markdown-stream 库在 AI 流式场景中的工程化落地、D3.js 原生力导向图实现技术关联可视化、NestJS SSE 流式推送架构、JWT 双令牌鉴权体系。

### 2.3 非目标

本项目不做以下功能：不提供岗位投递或简历投递功能；不做实时爬虫（数据为预先爬取后导入）；不做移动端原生适配（仅 Web 响应式）；不做多角色权限系统（仅普通用户角色）。

## 3. 用户角色与使用场景

### 3.1 用户角色

本系统仅有一种用户角色：求职者。未登录状态下可访问公开页面（首页、岗位列表、岗位详情），登录后可使用 AI 对话、个人仪表盘、收藏等功能。

### 3.2 核心使用场景

**场景一：市场行情速览**

小王刚开始准备跳槽，想了解当前前端市场整体情况。他打开 JobRadar 首页，看到当前在招前端岗位总数、平均薪资范围、热门技术栈排行、各城市岗位分布。他发现 TypeScript 的需求已经超过 JavaScript，决定在简历中重点突出 TS 能力。

**场景二：技术趋势洞察**

小李想知道"学 Vue 的话，还需要配套学什么"。他进入技术图谱页面，看到 Vue 节点周围连接着 TypeScript、Nuxt、Pinia、Vite 等节点，连线粗细表示共现频率。他点击 Vue 节点，高亮显示所有关联技术，发现"Vue + TypeScript"的共现率高达 78%，"Vue + Nuxt"的共现率也在快速上升。

**场景三：JD 深度分析**

小张在岗位列表中找到一个感兴趣的职位。他点击进入详情页，然后点击"AI 分析此岗位"按钮。AI 通过 SSE 流式输出分析结果：核心技术要求拆解（以技能雷达图展示）、隐含要求识别（以告警卡片提示）、薪资市场参考（以薪资对比条展示）。整个过程是逐字流式渲染的，图表和卡片在流中实时出现。

**场景四：技能差距诊断**

小赵在仪表盘中填入自己的技能栈，然后发起"技能匹配诊断"。AI 对比他的技能与目标岗位群体的要求，输出一份包含雷达图叠加对比的诊断报告，明确标出技能缺口区域和优先学习建议。

**场景五：模拟面试**

小钱准备面试字节跳动的前端岗位。他在 AI 对话中选择"模拟面试"场景并关联该岗位。AI 根据 JD 生成面试题，小钱回答后 AI 给出评价和参考答案。

## 4. 功能需求

### 4.1 模块总览

系统划分为六个功能模块：首页市场大盘、岗位浏览、技术图谱、AI 智能对话、个人仪表盘、用户鉴权。

### 4.2 M1 — 首页市场大盘

**页面路由**：`/`

**渲染模式**：SSR

**功能描述**：作为平台首屏入口，展示市场宏观数据概览，帮助用户快速感知当前求职市场整体态势。

**功能清单**：

F1.1 — 核心指标卡片：展示四个核心数据卡片，分别是在招岗位总数、前端岗位平均薪资（取中位数）、最热门技术 Top 1（出现频率最高的技能）、本周新增岗位数。数据从后端 `/api/analysis/market-overview` 接口获取。

F1.2 — 热门技术排行榜：展示 Top 20 热门技能及其出现频率，以水平柱状图形式呈现。支持按城市筛选。点击某技能可跳转至岗位列表页并自动带入该技能筛选条件。

F1.3 — 薪资分布图：展示前端岗位的薪资区间分布（如 10-15K 占比多少、15-20K 占比多少），以直方图或分布曲线呈现。支持按城市筛选。

F1.4 — 城市岗位分布：展示各城市的岗位数量占比，以饼图或横向柱状图呈现。点击城市可跳转至该城市的岗位列表。

F1.5 — 快速入口：提供"浏览岗位"、"技术图谱"、"AI 助手"三个功能入口卡片。

### 4.3 M2 — 岗位浏览

**页面路由**：`/jobs`（列表）、`/jobs/:id`（详情）

**渲染模式**：SSR

**功能描述**：提供岗位数据的检索、筛选、查看详情能力。

**功能清单**：

F2.1 — 岗位列表：以卡片列表形式展示岗位，每张卡片包含职位名称、公司名称、城市、薪资范围、经验要求、技能标签（取前 5 个）。默认按更新时间倒序排列。支持分页，每页 20 条。

F2.2 — 筛选面板：左侧或顶部提供多维度筛选，包括城市（多选）、薪资范围（滑块，如 0-100K）、经验要求（多选：不限/1-3年/3-5年/5-10年）、学历要求（多选）、技能标签（输入式多选，从已有技能中搜索）。筛选条件变化时 URL query 同步更新（支持分享筛选链接）。

F2.3 — 搜索：顶部搜索框，支持按职位名称和公司名称模糊搜索。

F2.4 — 排序：支持按薪资（高到低/低到高）、更新时间排序。

F2.5 — 岗位详情页：展示完整 JD 描述、技能标签列表（可点击跳转图谱）、薪资范围可视化条、公司信息、相似岗位推荐（基于技能重合度的 Top 5）。页面底部提供"AI 分析此岗位"按钮，点击后携带岗位 ID 跳转至 AI 对话页并自动发起分析。

F2.6 — 收藏功能：登录用户可收藏岗位，收藏状态在卡片和详情页均有体现。收藏列表在个人仪表盘中查看。需要登录。

### 4.4 M3 — 技术图谱

**页面路由**：`/graph`

**渲染模式**：CSR（`<ClientOnly>`）

**功能描述**：以 D3 力导向图的形式展示技术栈之间的关联关系，帮助用户理解"学 A 技术通常还需要配套学什么"。

**功能清单**：

F3.1 — 力导向图主视图：以 D3 力导向图展示技能节点和连线。节点代表一项技术（如 Vue、TypeScript、Node.js），节点大小与该技术在 JD 中的出现频率成正比，节点颜色按技术类别分组（前端框架/编程语言/工具链/后端/DevOps 各一个色系）。连线代表两项技术在同一 JD 中共同出现，连线粗细与共现次数成正比。

F3.2 — 交互行为：支持鼠标滚轮缩放和拖拽平移；节点可拖拽重新定位；hover 节点时显示 tooltip（技术名称、出现次数、关联技术数量、相关岗位平均薪资）；单击节点高亮该节点及其所有直接关联节点，其余节点半透明；双击技能节点跳转至岗位列表页，自动筛选包含该技能的岗位。

F3.3 — 控制面板：左侧或顶部提供控制面板，包括城市筛选（切换城市后图谱数据重新加载）、节点数量控制（Top N 滑块，范围 20-100，默认 50）、类别过滤（勾选/取消某个技术类别的显示）、搜索定位（输入技术名称，图谱自动缩放并居中到该节点）。

F3.4 — 图例：展示节点颜色与技术类别的对应关系，以及节点大小、连线粗细的含义说明。

F3.5 — 我的技能高亮：登录用户若已填写个人技能，图谱中对应节点以绿色描边高亮，直观看到自己掌握的技术在市场中的位置。需要登录。

### 4.5 M4 — AI 智能对话

**页面路由**：`/chat`

**渲染模式**：CSR（`<ClientOnly>`）

**功能描述**：提供多场景的 AI 对话能力，AI 输出通过 SSE 流式推送，前端使用自研的 vue-markdown-stream 库实时渲染为包含 Vue 组件块的富交互内容。需要登录。

**功能清单**：

F4.1 — 对话界面：左侧为历史会话列表（按时间倒序），右侧为当前对话内容区 + 底部输入框。对话内容区上方显示当前场景标识和关联的岗位信息（如有）。

F4.2 — 场景选择：对话页顶部或新建对话时，提供场景选择。五种场景分别是 JD 深度分析（需关联一个岗位）、技能匹配诊断（需用户已填写技能）、模拟面试（需关联一个岗位）、简历优化建议、自由问答。每种场景对应不同的后端 prompt 模板。

F4.3 — 流式渲染：AI 回复通过 SSE 逐字推送到前端，StreamRenderer 组件使用 vue-markdown-stream 实时渲染。在流式输出过程中，末尾显示光标闪烁动画（▍）。支持在流式输出过程中点击"停止生成"按钮中断 SSE 连接。

F4.4 — 组件块渲染：AI 在回复中可使用以下自定义 Markdown 组件块语法，vue-markdown-stream 将其实时渲染为对应的 Vue 组件。

F4.4.1 — `:::alert [type]`：类型包括 info（蓝色）、warning（橙色）、success（绿色）。渲染为带图标和背景色的提示卡片，内部支持 Markdown 格式文本。

F4.4.2 — `:::job-card {"title":"...","company":"...","salary":"...","city":"...","tags":[...]}`：渲染为一张职位信息卡片，展示职位名、公司、薪资、城市、技能标签。若 tags 超过 5 个则折叠显示。

F4.4.3 — `:::skill-radar {"labels":[...],"you":[...],"required":[...]}`：渲染为 D3 雷达图，叠加展示"你的技能"和"岗位要求"两组数据。"你的技能"用蓝色，"岗位要求"用红色，差值区域高亮。

F4.4.4 — `:::salary-bar {"min":N,"max":N,"median":N,"yourExpect":N}`：渲染为水平薪资对比条。显示岗位薪资范围（min-max）、市场中位数标记、用户期望薪资标记（如有）。

F4.4.5 — `:::compare`：内部包含 Markdown 表格，渲染为样式优化的对比表格，用于展示两个或多个岗位的横向对比。

F4.5 — 对话历史：每次对话自动保存到后端（chat_sessions 表）。用户可切换历史会话继续对话。支持删除某条历史会话。

F4.6 — 快捷入口：从岗位详情页点击"AI 分析此岗位"按钮跳转至对话页时，自动创建一个"JD 深度分析"场景的新对话，并关联该岗位 ID，AI 自动开始分析（无需用户手动输入第一条消息）。

F4.7 — 预设问题：输入框上方提供当前场景的预设问题标签（如 JD 分析场景提供"核心要求是什么"、"适合我吗"、"面试可能考什么"等），点击即自动填入并发送。

### 4.6 M5 — 个人仪表盘

**页面路由**：`/dashboard`

**渲染模式**：CSR（`<ClientOnly>`）

**功能描述**：登录用户的个人数据中心。需要登录。

**功能清单**：

F5.1 — 个人技能配置：用户可添加/删除自己掌握的技能标签（从系统技能库中搜索选择），设置目标城市和期望薪资。这些数据用于 AI 对话的"技能匹配诊断"场景和技术图谱的"我的技能高亮"功能。

F5.2 — 技能雷达图：基于用户填写的技能和市场需求数据，展示个人技能 vs 市场平均需求的 D3 雷达图对比。高亮技能缺口区域。

F5.3 — 收藏岗位列表：展示用户收藏的所有岗位，以卡片列表形式呈现，支持取消收藏和跳转详情。

F5.4 — AI 对话记录：展示用户的历史 AI 对话列表（按时间倒序），点击可跳转至对话页继续。

### 4.7 M6 — 用户鉴权

**页面路由**：`/auth/login`、`/auth/register`

**渲染模式**：SSR

**功能描述**：用户注册和登录。

**功能清单**：

F6.1 — 注册：邮箱 + 密码 + 昵称。密码要求至少 8 位，包含字母和数字。注册成功后自动登录并跳转首页。

F6.2 — 登录：邮箱 + 密码。登录成功后获取 Access Token（15 分钟有效）和 Refresh Token（7 天有效），存储在 httpOnly cookie 或 localStorage 中。

F6.3 — Token 自动刷新：Access Token 过期时，前端自动使用 Refresh Token 获取新的双令牌，用户无感知。

F6.4 — 登出：清除本地令牌，后端使 Refresh Token 失效（从 Redis 删除）。

F6.5 — 路由守卫：未登录用户访问需鉴权页面（/chat、/dashboard）时，重定向到登录页，并在登录后跳回原页面。

## 5. 非功能需求

### 5.1 性能

SSR 页面首屏 LCP 控制在 1.5 秒以内。D3 力导向图在 100 个节点规模下交互流畅（60fps）。SSE 流式输出延迟从用户发送到首字符出现控制在 2 秒以内（取决于大模型响应速度）。

### 5.2 兼容性

浏览器支持 Chrome 90+、Firefox 90+、Safari 15+、Edge 90+。响应式适配桌面端（1200px+）和平板端（768px+），移动端不做特别优化但保证基本可用。

### 5.3 安全

密码使用 bcrypt 加盐哈希存储。JWT Access Token 短时效（15 分钟）降低泄露风险。Refresh Token 存 Redis 支持服务端主动失效。API 接口统一鉴权，未授权请求返回 401。AI 对话接口做频率限制，防止滥用。

### 5.4 可部署性

提供 Docker Compose 配置，任何人执行 `docker compose up` 即可启动完整服务（前端 + 后端 + PostgreSQL + Redis）。提供 `.env.example` 文件说明所有需要配置的环境变量。提供 README 包含项目介绍、架构说明、本地运行指南、截图演示。

## 6. 页面与信息架构

### 6.1 页面地图

```
JobRadar
├── 首页 /                          [公开] [SSR]
├── 岗位
│   ├── 岗位列表 /jobs              [公开] [SSR]
│   └── 岗位详情 /jobs/:id          [公开] [SSR]
├── 技术图谱 /graph                  [公开] [CSR]
├── AI 对话 /chat                    [需登录] [CSR]
├── 个人仪表盘 /dashboard            [需登录] [CSR]
└── 鉴权
    ├── 登录 /auth/login             [公开] [SSR]
    └── 注册 /auth/register          [公开] [SSR]
```

### 6.2 全局布局

所有页面共享顶部导航栏（Logo + 导航菜单 + 登录/用户头像）和底部页脚（版权信息 + GitHub 链接）。导航菜单包含：首页、岗位、图谱、AI 助手、仪表盘。未登录时"AI 助手"和"仪表盘"显示为带锁标识，点击跳转登录页。

### 6.3 核心用户流程

**流程一：浏览 → 分析 → 诊断**

用户在首页看到市场概览 → 点击进入岗位列表筛选目标岗位 → 进入某岗位详情 → 点击"AI 分析此岗位" → 跳转至 AI 对话页，AI 自动输出包含组件块的分析报告 → 用户在对话中继续追问 → 跳转仪表盘查看技能差距。

**流程二：图谱探索 → 发现 → 学习**

用户进入技术图谱 → 找到自己掌握的技术节点（绿色高亮） → 发现关联的高频共现技术 → 双击该技术跳转岗位列表查看具体岗位 → 决定下一步学习方向。

**流程三：对话 → 准备 → 面试**

用户在 AI 对话页选择"模拟面试"场景 → 关联目标岗位 → AI 生成面试题 → 用户回答 → AI 评价并给出参考答案 → 用户反复练习。

## 7. API 接口清单

### 7.1 鉴权

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| POST | /api/auth/register | 注册 | 否 |
| POST | /api/auth/login | 登录 | 否 |
| POST | /api/auth/refresh | 刷新令牌 | Refresh Token |
| POST | /api/auth/logout | 登出 | Access Token |

### 7.2 岗位

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| GET | /api/jobs | 岗位列表（筛选+分页） | 否 |
| GET | /api/jobs/:id | 岗位详情 | 否 |
| GET | /api/jobs/:id/similar | 相似岗位推荐 | 否 |
| POST | /api/jobs/:id/favorite | 收藏岗位 | 是 |
| DELETE | /api/jobs/:id/favorite | 取消收藏 | 是 |
| GET | /api/jobs/favorites | 我的收藏列表 | 是 |

## 8. 数据说明

### 8.1 数据来源

岗位数据来源于 Boss 直聘平台的公开岗位信息，通过预先爬取后批量导入数据库。系统不做实时爬虫，数据通过 `scripts/seed.ts` 脚本导入。

### 8.2 数据规模

预期初始数据量约 5000-10000 条前端相关岗位，覆盖一线及新一线城市（北京、上海、广州、深圳、杭州、成都、南京、武汉等）。技能共现矩阵约 100-200 个技能节点。

### 8.3 技能提取

从 JD 文本中提取技能标签的逻辑基于关键词词典匹配，词典覆盖约 80 个前端相关技术（框架、语言、工具链、后端、DevOps）。提取结果存储在 jobs 表的 skills 数组字段中。

### 8.4 技能共现矩阵

通过 `scripts/build-cooccurrence.ts` 脚本计算：遍历所有岗位的 skills 数组，对每对技能组合计数，结果存入 skill_cooccurrence 表。该表作为 D3 技术图谱的数据源。
